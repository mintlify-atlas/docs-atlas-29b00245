title: "Quickstart"
description: "Get EntityModelBinder working in your ASP.NET Core project in minutes"
---

This guide walks through a complete working example from package install to a fully functional controller with automatic route model binding.

<Steps>
  <Step title="Install the package">
    Add EntityModelBinder to your ASP.NET Core project.

    ```bash
    dotnet add package EntityModelBinder
    ```
  </Step>

  <Step title="Implement IEntity on your model">
    Implement the `IEntity` marker interface on any entity class you want to resolve from routes.

    ```csharp Product.cs
    using EntityModelBinder;

    public class Product : IEntity
    {
        public int Id { get; set; }
        public string Slug { get; set; } = string.Empty;
        public string Name { get; set; } = string.Empty;
        public string Description { get; set; } = string.Empty;
    }
    ```

    <Note>
      `IEntity` has no required members. It is a marker interface that tells EntityModelBinder which types are eligible for automatic database resolution.
    </Note>
  </Step>

  <Step title="Register EntityModelBinder in Program.cs">
    Call `AddEntityModelBinder<TContext>()` with your EF Core `DbContext`. This registers the model binder provider and replaces the standard `AddControllers()` call.

    ```csharp Program.cs
    using EntityModelBinder;
    using Microsoft.EntityFrameworkCore;

    var builder = WebApplication.CreateBuilder(args);

    builder.Services.AddDbContext<AppDbContext>(options =>
        options.UseSqlServer(builder.Configuration.GetConnectionString("Default")));

    builder.Services.AddEntityModelBinder<AppDbContext>();

    var app = builder.Build();

    app.UseRouting();
    app.MapControllers();
    app.Run();
    ```
  </Step>

  <Step title="Use [FromRoute] in your controller">
    Declare route-bound entities as `[FromRoute]` parameters. EntityModelBinder resolves them from EF Core using the route value matching the primary key column.

    ```csharp ProductsController.cs
    using Microsoft.AspNetCore.Mvc;

    [ApiController]
    [Route("api/[controller]")]
    public class ProductsController : ControllerBase
    {
        private readonly AppDbContext _context;

        public ProductsController(AppDbContext context)
        {
            _context = context;
        }

        // GET /api/products/42
        [HttpGet("{id}")]
        public IActionResult Get([FromRoute] Product product)
        {
            return Ok(product);
        }

        // POST /api/products
        [HttpPost]
        public async Task<IActionResult> Create([FromBody] Product product)
        {
            _context.Products.Add(product);
            await _context.SaveChangesAsync();
            return CreatedAtAction(nameof(Get), new { id = product.Id }, product);
        }

        // PATCH /api/products/42
        [HttpPatch("{id}")]
        public async Task<IActionResult> Update(
            [FromRoute] Product product,
            [FromBody] ProductUpdateRequest request)
        {
            product.Name = request.Name ?? product.Name;
            product.Description = request.Description ?? product.Description;

            await _context.SaveChangesAsync();
            return Ok(product);
        }
    }
    ```

    Requests to `GET /api/products/42` will have the `Product` with `Id = 42` injected directly into the action. If no product exists with that ID, EntityModelBinder returns `404 Not Found` before your action is invoked.
  </Step>

  <Step title="Optional: bind by a custom key column">
    By default, EntityModelBinder looks up entities by the `Id` column. To bind by a different column — such as a `Slug` — use the `[EntityKey]` attribute on the parameter.

    ```csharp ProductsController.cs
    using EntityModelBinder;

    // GET /api/products/my-product-slug
    [HttpGet("{slug}")]
    public IActionResult GetBySlug([FromRoute][EntityKey("Slug")] Product product)
    {
        return Ok(product);
    }
    ```

    The `[EntityKey("Slug")]` attribute instructs the binder to query `WHERE Slug = {slug}` instead of the default primary key lookup.

    <Tip>
      `[EntityKey]` accepts any column name that exists on the entity. You can use it alongside `[FromRoute]` on the same parameter.
    </Tip>
  </Step>
</Steps>

<Check>
  You now have automatic route model binding working in your ASP.NET Core project. Incoming route values are resolved to EF Core entities, and missing records return 404 automatically — no manual lookups required.
</Check>
