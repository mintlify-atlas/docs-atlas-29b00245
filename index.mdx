## What is EntityModelBinder?

EntityModelBinder brings **Laravel's route model binding** pattern to ASP.NET Core. Instead of manually fetching entities in every controller action, you declare the entity as a route parameter and it's resolved automatically from your Entity Framework Core database context.

**Before** — repetitive boilerplate in every action:

```csharp
[HttpGet("{surveyId:int}")]
public async Task<IActionResult> Show([FromRoute] int surveyId)
{
    var survey = await _dbContext.Survey.FindAsync(surveyId);
    if (survey == null) return NotFound();
    return Ok(survey);
}
```

**After** — clean, declarative controllers:

```csharp
[HttpGet("{survey}")]
public IActionResult Show([FromRoute] Survey survey)
    => survey == null ? NotFound() : Ok(survey);
```

## Requirements

- .NET 8.0 or later (also supports .NET 9 and .NET 10)
- ASP.NET Core
- Entity Framework Core 8.0 or later
